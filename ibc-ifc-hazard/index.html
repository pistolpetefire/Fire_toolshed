IBC / IFC Hazard Impacts — Help / Guide

What this is: a preliminary requirements checklist for areas with Group H and/or hazardous materials. It unions construction and systems items commonly driven by IBC Ch. 3 (H-1…H-5), §414 / §415, high-piled context (§315 / IFC 32), and IFC Ch. 50+ material chapters. You can fill selections manually or import an HMIS inventory spreadsheet to seed the form.

What this is not: an MAQ calculator, SDS validator, control-area floor plan tool, or AHJ determination. Quantities, separations, and local amendments decide occupancy — this tool maps your stated path and hazard set to typical mitigations.

Recommended workflow

  1. Import (optional) — §0 · upload HMIS .xlsx / .csv → review assessment → Apply to form. Always re-check every box after apply.
  2. Edition — match the adopted IBC/IFC pair (2015–2024). Import may detect a year from the file; confirm it.
  3. MAQ path — Group H (above MAQ) vs control areas only. This choice changes which drivers fire.
  4. Hazards — every IFC material family present or planned (import seeds these from classification text).
  5. H groups — confirm Chapter 3 classifications; use suggested chips if anything is still unchecked.
  6. High-piled — only if storage meets high-piled criteria (not set by import).
  7. Filters — optional high-cost only or single category.
  8. Table + report — review drivers; Print to PDF or Save HTML for design meetings.

Import HMIS inventory

Section 0 · Import HMIS inventory reads a materials list in the browser and proposes form values. Nothing is sent to a server.

What to upload

  • .xlsx (preferred) or .csv export of an HMIS / hazmat inventory.
  • Must include a header row with HAZARD CLASSIFICATION (or close aliases: Hazard Class, Classification, IFC Class).
  • Strongly recommended: PRODUCT NAME. Optional: Component, CAS, Concentration, Storage location, Container size, Stored quantity, Product code.
  • Workbooks may include extra sheets. The app prefers inventory-like sheet names (SDS, HMIS, inventory, materials) and ignores pure notes sheets when possible.

How assessment works

  1. Finds the inventory header and groups multi-row products (one product name + component lines, as in typical HMIS layouts).
  2. Collects every non-blank hazard classification string on those rows.
  3. Keyword-matches each string to IFC hazard families used by this app (e.g. “FLAMMABLE LIQUID CLASS IB” → flammable liquids Class I; “CLASS 2 OXIDIZER” → oxidizer; “ORGANIC PEROXIDE CLASS IV” → organic peroxides; “CORROSIVE CAT 1A” → corrosive).
  4. Unions matched families → hazard checkboxes; maps each family to Chapter 3 suggested H groups.
  5. If a MAQ TABLES-style sheet has MAQ EXCEEDED = YES / NO / TBD, suggests a MAQ path (see table below).
  6. May fill project name, facility hints, and edition when found in titles or MAQ references (e.g. “2024 IFC”).
MAQ EXCEEDED in file Suggested path Confidence
Any YES Group H (above MAQ) High — still confirm quantities and control areas.
All NO Control areas only High — still confirm MAQ tables and separations.
TBD / missing Group H (conservative default) Low — engineer must decide; import does not calculate MAQs.

MAQ path — choose deliberately

Path When to use What the matrix does
Group H
(above MAQ)		 Area is classified (or expected to be) H-1…H-5. Materials exceed MAQ and control-area strategy is not the design intent. H-group checkboxes and hazards drive the list. Includes full H construction packages: type of construction, roof Class A, fire walls/barriers, H systems.
Control areas
(at / under MAQ)		 Screening “can we stay out of Group H?” or design intent is control areas with quantities managed under MAQ. H-group construction drivers are off. Only selected hazards and high-piled fire items. Emphasizes §414 control areas, separation, spill/exhaust, IFC use controls. H checkboxes are ignored as drivers (suggestions still help if you flip path later).

Neither path computes MAQ tables or number of control areas by story. That remains a code study with IFC Table 5003.1.1(1)/(2) (or edition equivalent) and IBC 414.

Inputs — field guide

IBC / IFC edition
Select the code year your project adopts. Core H definitions and 414/415 structure are stable across 2015–2024; always re-check tables and local amendments. Import may set this when a year appears in the workbook (e.g. MAQ sheet cite).
Project name / area
Labels the print package. Prefer a specific suite, process bay, or warehouse zone. Import may fill project from HMIS title text and facility from storage location / title hints (template leftovers in a file should be overwritten).
Building sprinklered
Context for the report and for items that assume NFPA 13. Not changed by import. Unsprinklered buildings need extra AHJ scrutiny on H and high-piled paths.
Report logo
Benham, Haskell, or upload — appears on Print / Save packages only.
H occupancy (H-1 … H-5)
IBC Chapter 3 classifications for the analysis area.
  • H-1 — detonation (e.g. explosives, severe unstable / organic peroxide classes).
  • H-2 — deflagration / accelerated burning (flammable gas, open Class I use, certain dusts, etc.).
  • H-3 — materials that readily support combustion / physical hazard (combustibles, many oxidizers, aerosols…).
  • H-4 — health hazard (highly toxic, toxic, corrosives above H thresholds).
  • H-5 — semiconductor HPM fabrication.
Multiple groups are allowed; requirements are unioned. Import can apply or merge suggested H from hazard maps.
Suggested H from hazards
Each hazard type carries a Chapter 3 map (shown as “often H-x” under the hazard). Unchecked mapped groups appear as chips and highlighted cards. Apply one or Apply all after manual edits; import may already have applied them depending on the H option you chose.
Material / hazard types
IFC class families (Ch. 50+). Check everything present or planned. Same mitigation from several hazards appears once with every driver listed. Hazard selection works on both MAQ paths; on the control-area path it is the primary driver set (plus high-piled).
High-piled combustible storage
When storage height/commodity meets high-piled criteria (IFC 32 / IBC §315 context). Adds high-piled and smoke/heat-vent style items even with no H group. Not set by HMIS import. Commodity class and ESFR depth are not modeled.
Filters (Show / Category)
High-cost construction only keeps budget drivers (walls, roof class, explosion control, specialized exhaust, containment, full H packages). Category focus narrows to construction, separation, explosion, mechanical, fire protection, or operational/IFC controls.

Reading the results table

  • Requirement — mitigation in plain language + short design note.
  • Category — construction, separation, explosion, mechanical, fire protection, or operational.
  • Code anchors — starting sections, not a complete citation tree. Open the adopted book.
  • Driven by — which H group(s), hazard label(s), or “High-piled storage” triggered the row. Multiple drivers mean one shared requirement.
  • HIGH COST — typical capital/program cost drivers for early budgeting; not a dollar estimate.

Summary panel shows edition, MAQ path, selected H, H mapped from hazards, hazard count, high-piled, and high-cost tally. If the control-area path shows zero rows, select hazards or high-piled — H checkboxes alone do not drive that path.

Merging rules (multi-hazard)

  • Requirements are the union of all matching triggers.
  • Duplicate IDs collapse to one row; all drivers are listed.
  • There is no silent “severity wins and deletes lesser rows” pass — if two different construction items both match, both appear.

Outputs

  • Print to PDF — opens a print dialog; choose Save as PDF. Cancel returns here.
  • Save report — downloads a standalone HTML package (logo + matrix) for email or archive.
  • Reset — clears saved selections and dismisses any import preview for this tool only.

Limitations — do not skip

  • No MAQ arithmetic, no control-area count by floor, no outdoor tank farm separation distances calculated.
  • HMIS import uses keyword matching on classification text only — it does not compute MAQs from gallons/pounds, parse full SDS sections, or prove occupancy.
  • Unmatched or unusual classification wording will not light hazards until you add them manually or reword the sheet.
  • Edition differences are footnotes, not a full 2015↔2024 delta set for every table.
  • Local amendments, risk engineering, and AHJ policy override this matrix.
  • Explosion control rows point to NFPA 68/69 (and related Toolshed tools) — they do not size vents.
  • Always verify against the adopted code text and the designer of record before design or permit.
